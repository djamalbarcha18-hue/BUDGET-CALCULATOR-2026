# ورقة "الإعدادات وأسعار الصرف"
## المخطط المعماري للطبقة التأسيسية لنظام مالي ذكي متكامل

> **اسم الورقة في المصنّف (Sheet Name):** `الإعدادات وأسعار الصرف`
> **الدور الوظيفي:** طبقة الإعداد المركزية التي تُغذّي كافة الأوراق الشهرية الاثنتي عشرة (يناير → ديسمبر) بمحرّك العملات، فئات الدخل والمصاريف، طرق الدفع، والتنسيقات المالية.
> **مصدر الحقيقة الوحيد (Single Source of Truth):** كل قائمة منسدلة، ودالة تحويل، وتنسيق رقمي في المصنّف يجب أن يُشير إلى نطاقات هذه الورقة عبر النطاقات المُسماة (Named Ranges) فقط، لضمان قابلية الصيانة والتوسع.

---

## 1. التخطيط الخلوي التفصيلي (Cell-by-Cell Layout)

التخطيط أدناه يُمثّل البنية المرجعية الكاملة لورقة "الإعدادات وأسعار الصرف". الأعمدة `A:D` مخصّصة لمحرّك العملات، والأعمدة `F`، `G`، `H` لقوائم المدخلات (الدخل، المصاريف، طرق الدفع).

### 1.1 ترويسة الورقة وعناصر التحكم العامة

| الخلية | المحتوى | الملاحظات |
|---|---|---|
| `A1` | `نظام مالي ذكي متكامل - الإعدادات وأسعار الصرف` | عنوان رئيسي (يُدمج عبر `A1:D1`، خط بارز). |
| `A2` | `يجب تحديث أسعار الصرف يدوياً أو عبر دالة GOOGLEFINANCE قبل بدء كل شهر مالي.` | تنبيه نصي (يُدمج عبر `A2:D2`). |
| `A3` | `العملة الرئيسية للعرض` | تسمية. |
| `B3` | `USD` (افتراضي) | **خلية اختيار العملة الرئيسية** - ترتبط بـ Data Validation إلى `rng_Currencies`. هذه الخلية هي محور المصنّف بالكامل. |
| `A4` | `تنسيق العملة النشط` | تسمية. |
| `B4` | `=XLOOKUP(B3, rng_Currencies, rng_FormatStrings)` | يعيد سلسلة التنسيق الرقمي الموافقة للعملة الرئيسية، وتُستخدم كمصدر مرجعي وحيد للتنسيق في كل الأوراق الشهرية. |
| `A5` | `سعر صرف العملة الرئيسية مقابل الدولار` | تسمية. |
| `B5` | `=XLOOKUP(B3, rng_Currencies, rng_CurrencyRates)` | معامل التحويل الفعّال للعملة الرئيسية. |

### 1.2 جدول محرّك العملات (الصف 6 = ترويسة، الصفوف 7→20 = البيانات)

| النطاق | المحتوى |
|---|---|
| `A6:D6` | ترويسة الجدول: `رمز العملة` ، `اسم العملة` ، `سعر الصرف مقابل الدولار USD` ، `التنسيق المالي الافتراضي` |
| `A7:A20` | رموز العملات (14 صفاً) |

| `D7:D20` | سلسلة التنسيق المالي (Number Format String) |

### 1.3 جداول قوائم المدخلات

| النطاق | المحتوى |
|---|---|
| `F6` | ترويسة: `فئات الدخل` |
| `F7:F14` | عناصر الدخل (8 عناصر) |
| `G6` | ترويسة: `فئات المصاريف` |

| `H7:H10` | طرق الدفع (4 عناصر) |

## 2. محرّك العملات المتعددة (Currency Engine)

### 2.1 جدول العملات الأساسي

البيانات أدناه تُمثّل المحتوى الذي يجب لصقه في `A7:D20`. الأسعار **إرشادية تقريبية حديثة** ويجب على المستخدم تحديثها قبل التشغيل الفعلي (إمّا يدوياً، أو ديناميكياً عبر دالة `GOOGLEFINANCE("CURRENCY:USD"&A7)` ضمن العمود `C`).

| رمز العملة | اسم العملة | سعر الصرف مقابل الدولار USD | التنسيق المالي الافتراضي |
|---|---|---|---|
| `DZD` | الدينار الجزائري | `134.50` | `#,##0.00 "د.ج"` |
| `SAR` | الريال السعودي | `3.7500` | `#,##0.00 "ر.س"` |
| `AED` | الدرهم الإماراتي | `3.6725` | `#,##0.00 "د.إ"` |
| `QAR` | الريال القطري | `3.6400` | `#,##0.00 "ر.ق"` |
| `KWD` | الدينار الكويتي | `0.3100` | `#,##0.000 "د.ك"` |
| `BHD` | الدينار البحريني | `0.3760` | `#,##0.000 "د.ب"` |
| `OMR` | الريال العماني | `0.3845` | `#,##0.000 "ر.ع"` |
| `EGP` | الجنيه المصري | `48.0000` | `#,##0.00 "ج.م"` |
| `JOD` | الدينار الأردني | `0.7090` | `#,##0.00 "د.أ"` |
| `TND` | الدينار التونسي | `3.1500` | `#,##0.000 "د.ت"` |
| `MAD` | الدرهم المغربي | `9.9500` | `#,##0.00 "د.م"` |
| `EUR` | اليورو | `0.9200` | `#,##0.00 €` |
| `USD` | الدولار الأمريكي | `1.0000` | `#,##0.00 $` |
| `GBP` | الجنيه الإسترليني | `0.7900` | `#,##0.00 £` |

> **اصطلاح القراءة:** سعر الصرف في العمود `C` يُمثّل **عدد الوحدات من العملة المحلية المقابلة لدولار أمريكي واحد**. وعليه فإن `USD = 1.0000` بطبيعة الحال، ويتم استخدام الدولار كعملة جسر (Bridge Currency) في كل التحويلات المتقاطعة.

### 2.2 منطق التحويل (USD كعملة جسر)

لتحويل مبلغ `Amount` من عملة المعاملة `Cur_T` إلى العملة الرئيسية للعرض `Cur_M`:

```
Amount_in_USD     = Amount / Rate(Cur_T)
Amount_in_Display = Amount_in_USD × Rate(Cur_M)
                  = Amount × Rate(Cur_M) / Rate(Cur_T)
```

### 2.3 دالة استدعاء سعر الصرف (XLOOKUP)

**نسخة المرجع المباشر:**

```
=XLOOKUP(A7, A7:A20, C7:C20)
```

**نسخة النطاقات المُسمّاة (المعتمدة في كل المصنّف):**

```
=XLOOKUP("SAR", rng_Currencies, rng_CurrencyRates)
```

### 2.4 صيغة التحويل الديناميكية الموحّدة

افتراض: في الأوراق الشهرية، يحوي العمود `D` المبلغ، والعمود `E` رمز عملة المعاملة (مثلاً `EUR`).

**النسخة بالمراجع المباشرة (داخل ورقة شهرية، الصف الحالي رقم 2):**

```
=D2 * XLOOKUP('الإعدادات وأسعار الصرف'!$B$3, 'الإعدادات وأسعار الصرف'!$A$7:$A$20, 'الإعدادات وأسعار الصرف'!$C$7:$C$20) / XLOOKUP(E2, 'الإعدادات وأسعار الصرف'!$A$7:$A$20, 'الإعدادات وأسعار الصرف'!$C$7:$C$20)
```

**النسخة الموصى بها بالنطاقات المُسمّاة (أنظف وأقصر):**

```
=D2 * XLOOKUP(rng_MainCurrency, rng_Currencies, rng_CurrencyRates) / XLOOKUP(E2, rng_Currencies, rng_CurrencyRates)
```

**نسخة `LET` المُحسَّنة للأداء (متوافقة مع Google Sheets):**

```
=LET(
   amt,      D2,
   txnCur,   E2,
   rateMain, XLOOKUP(rng_MainCurrency, rng_Currencies, rng_CurrencyRates),
   rateTxn,  XLOOKUP(txnCur,           rng_Currencies, rng_CurrencyRates),
   IFERROR( amt * rateMain / rateTxn, 0 )
)
```

### 2.5 صيغة `ARRAYFORMULA` لتحويل عمود معاملات بأكمله دفعة واحدة

تُكتب مرة واحدة في الخلية `F2` من الورقة الشهرية، وتغطّي كامل العمود تلقائياً:

```
=ARRAYFORMULA(
   IF( D2:D = "" , "" ,
       D2:D
       * XLOOKUP(rng_MainCurrency, rng_Currencies, rng_CurrencyRates)
       / IFERROR( XLOOKUP(E2:E, rng_Currencies, rng_CurrencyRates), 1 )
   )
)
```

### 2.6 صيغة استرجاع تنسيق العملة النشط

تُكتب في `B4` من ورقة الإعدادات، ويُستفاد منها كمصدر مرجعي وحيد للتنسيق الرقمي عبر سكربت `Apps Script` أو عبر `TEXT()`:

```
=XLOOKUP(rng_MainCurrency, rng_Currencies, rng_FormatStrings)
```

لعرض مبلغ مُحوَّل بسلسلة التنسيق الفعّالة في خلية نصية:

```
=TEXT( F2 , rng_ActiveFormat )
```

## 4. النطاقات المُسمّاة (Named Ranges Configuration)

> **القاعدة:** كل صيغة في كل ورقة شهرية تستدعي البيانات عبر النطاقات المُسمّاة فقط - لا مراجع مباشرة بأرقام صفوف. هذا يضمن أن أي توسيع للجداول مستقبلاً (إضافة عملات أو فئات) لا يكسر أيّ صيغة.

| اسم النطاق | المرجع `A1` الكامل | الغرض |
|---|---|---|
| `rng_MainCurrency` | `'الإعدادات وأسعار الصرف'!$B$3` | خلية اختيار العملة الرئيسية للعرض. |
| `rng_ActiveFormat` | `'الإعدادات وأسعار الصرف'!$B$4` | سلسلة تنسيق العملة النشطة (مُشتقّة). |
| `rng_MainRate` | `'الإعدادات وأسعار الصرف'!$B$5` | سعر صرف العملة الرئيسية مقابل الدولار. |
| `rng_Currencies` | `'الإعدادات وأسعار الصرف'!$A$7:$A$20` | عمود رموز العملات (مفتاح البحث). |
| `rng_CurrencyNames` | `'الإعدادات وأسعار الصرف'!$B$7:$B$20` | أسماء العملات بالعربية. |
| `rng_CurrencyRates` | `'الإعدادات وأسعار الصرف'!$C$7:$C$20` | أسعار الصرف مقابل USD. |
| `rng_FormatStrings` | `'الإعدادات وأسعار الصرف'!$D$7:$D$20` | سلاسل التنسيق المالي لكل عملة. |
| `rng_CurrencyTable` | `'الإعدادات وأسعار الصرف'!$A$7:$D$20` | الجدول الكامل لمحرّك العملات. |
| `rng_IncomeCategories` | `'الإعدادات وأسعار الصرف'!$F$7:$F$14` | قائمة فئات الدخل. |
| `rng_ExpenseCategories` | `'الإعدادات وأسعار الصرف'!$G$7:$G$18` | قائمة فئات المصاريف (12 عنصراً). |
| `rng_PaymentMethods` | `'الإعدادات وأسعار الصرف'!$H$7:$H$10` | قائمة طرق الدفع. |

**خطوات إنشاء النطاق المُسمّى في Google Sheets:**
`Data → Named ranges → + Add a range → الاسم + المرجع → Done`.

## 5. قواعد التحقق من البيانات (Data Validation Rules) للأوراق الشهرية

تُطبَّق القواعد التالية على أعمدة الإدخال في كلٍّ من الأوراق الشهرية الاثنتي عشرة. تخطيط الأعمدة المرجعي للورقة الشهرية:
`A: التاريخ` ، `B: نوع المعاملة (دخل/مصروف)` ، `C: الفئة` ، `D: المبلغ بعملة المعاملة` ، `E: عملة المعاملة` ، `F: المبلغ بعملة العرض` ، `G: طريقة الدفع` ، `H: ملاحظات`.

| العمود | نوع التحقق | المصدر |
|---|---|---|
| `C` (الفئة - للدخل) | List from a range | `=rng_IncomeCategories` |
| `C` (الفئة - للمصاريف) | List from a range | `=rng_ExpenseCategories` |
| `E` (عملة المعاملة) | List from a range | `=rng_Currencies` |
| `G` (طريقة الدفع) | List from a range | `=rng_PaymentMethods` |
| `D` (المبلغ) | Number > 0 | تحقق رقمي موجب. |

> **ملاحظة بشأن قائمة الفئة المشروطة:** للحصول على قائمة فئات تتغيّر تلقائياً بحسب نوع المعاملة (دخل أو مصروف) في العمود `B`، يُستخدم عمود مساعد بصيغة:
>
> ```
> =IF(B2="دخل", FILTER(rng_IncomeCategories, rng_IncomeCategories<>""),
>     IF(B2="مصروف", FILTER(rng_ExpenseCategories, rng_ExpenseCategories<>""), ""))
> ```
>
> ثم يُربط Data Validation للعمود `C` بهذا العمود المساعد.

## 6. آلية ربط الأوراق الشهرية الاثنتي عشرة بطبقة الإعدادات

ستُنشأ لاحقاً 12 ورقة شهرية بالأسماء التالية بترتيب التقويم الميلادي:

> `يناير` ، `فبراير` ، `مارس` ، `أبريل` ، `مايو` ، `يونيو` ، `يوليو` ، `أغسطس` ، `سبتمبر` ، `أكتوبر` ، `نوفمبر` ، `ديسمبر`.

كل ورقة شهرية ترتبط بطبقة الإعدادات حصراً عبر النطاقات المُسمّاة، ولا تحوي أيّ قائمة بيانات مرجعية محلية. القواعد الإلزامية:

1. **العملة الرئيسية:** كل عرض إجمالي في الورقة الشهرية يستخدم `rng_MainCurrency` لتحديد عملة العرض. تغيير الخلية `B3` في ورقة الإعدادات يُحدّث كل الأوراق الشهرية فوراً.
2. **التحويل:** كل صف معاملة يستخدم صيغة التحويل الموحّدة (الفقرة 2.4) أو نسخة `ARRAYFORMULA` (الفقرة 2.5) لإنتاج عمود "المبلغ بعملة العرض" من عمودَي "المبلغ بعملة المعاملة" و"عملة المعاملة".
3. **القوائم المنسدلة:** كل خلية إدخال للفئة وعملة المعاملة وطريقة الدفع تشير إلى النطاق المُسمّى المقابل (الفقرة 5).
4. **التنسيق المالي:** يُطبَّق `rng_ActiveFormat` على كل عمود مالي عبر `Format → Number → Custom number format` - أو ديناميكياً عبر `=TEXT(value, rng_ActiveFormat)` عند الحاجة لعرض نصي.
5. **منع الكتابة فوق البيانات المرجعية:** ورقة "الإعدادات وأسعار الصرف" يجب حمايتها (`Data → Protect sheet`) بحيث لا تُعدَّل سوى الخلايا `B3` (العملة الرئيسية) و`C7:C20` (تحديث الأسعار) والقوائم في `F7:H18`.
6. **المخطط الموحّد للأوراق الشهرية:** ينبغي أن تنسخ كل ورقة شهرية نفس مخطط الأعمدة `A:H` المذكور في الفقرة 5، بحيث يصبح من الممكن لاحقاً بناء ورقة "الملخّص السنوي" عبر `QUERY` أو `=SUMIFS` مع ضمّ النطاقات (Stacking).

### 6.1 مثال تطبيقي كامل لصف واحد في ورقة `يناير`

| العمود | الصيغة / القيمة |
|---|---|
| `A2` | `2026-01-05` |
| `B2` | `مصروف` |
| `C2` | `الطعام` *(من `rng_ExpenseCategories`)* |
| `D2` | `350` |
| `E2` | `SAR` *(من `rng_Currencies`)* |
| `F2` | `=D2 * XLOOKUP(rng_MainCurrency, rng_Currencies, rng_CurrencyRates) / XLOOKUP(E2, rng_Currencies, rng_CurrencyRates)` |
| `G2` | `بطاقة بنكية` *(من `rng_PaymentMethods`)* |
| `H2` | `سوبرماركت أسبوعي` |

أو على مستوى العمود بأكمله، تُكتب في `F2` فقط:

```
=ARRAYFORMULA(
   IF(D2:D="", "",
      D2:D
      * XLOOKUP(rng_MainCurrency, rng_Currencies, rng_CurrencyRates)
      / IFERROR( XLOOKUP(E2:E, rng_Currencies, rng_CurrencyRates), 1 )
   )
)
```

## 7. ملاحظات هندسية ختامية

- **الأسعار الإرشادية:** القيم المُدرَجة في العمود `C` هي مؤشرات تقريبية فقط، يلزم تحديثها قبل أيّ استخدام محاسبي فعلي.
- **التحديث الآلي للأسعار (اختياري):** يمكن استبدال أيّ خلية في `C7:C20` بصيغة:

  ```
  =IFERROR( GOOGLEFINANCE("CURRENCY:USD"&A7) , <last_known_rate> )
  ```

  مع ملاحظة أن `GOOGLEFINANCE` لا يدعم بعض العملات الإقليمية بشكل دائم، لذا يُنصح بالاحتفاظ بآخر سعر معروف كقيمة احتياطية.
- **التوسعة المستقبلية:** إضافة عملة جديدة تتمّ بإلحاق صف داخل `A7:D20` ثم توسيع النطاقات المُسمّاة بحدوده الجديدة - كل الصيغ ستلتقط الإضافة تلقائياً.
- **الاتساق الاصطلاحي:** كل الواجهات الظاهرة للمستخدم (الترويسات، أسماء الفئات، الرسائل) تتقيّد بمصطلحات مالية عربية مهنية موحّدة، بما يضمن وضوح التقارير ومخرجات الميزانية.
